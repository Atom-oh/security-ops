Section 01

Executive Summary

2026년 4월, Anthropic은 Claude Mythos Preview를 발표하며 애플리케이션 보안의 새로운 시대를 열었습니다. Mythos는 단순 모델이 아닌 "Model + Scaffold = Agent" 구조로, 기존 Rule-based/LLM-based 보안을 넘어 완전한 Agent-based 보안의 시대를 예고합니다.

Section 02

Application Security의 진화

보안 도구의 진화는 3단계로 구분됩니다. Mythos는 Phase 3 Agent-based의 선두주자입니다.

1990s ~ 2020s
Phase 1. Rule-based
SAST → DAST → 코드 리뷰

정규식 패턴 매칭, 시그니처 기반 탐지. 알려진 취약점만 탐지 가능. 높은 오탐률.

2023 ~ 2025
Phase 2. LLM-based
SAST + LLM → DAST + LLM

LLM이 코드 의미를 이해. 컨텍스트 기반 분석. 하지만 단발성 질의-응답 구조의 한계.

2026 ~
Phase 3. Agent-based
Google Naptime → Anthropic Mythos

자율 에이전트가 탐색-발견-검증-익스플로잇 전 과정을 인간 개입 없이 수행. 재귀적 추론.

💡 Phase 3의 핵심 차별점

Agent-based는 단순히 "LLM이 더 똑똑해진 것"이 아닙니다. 자율적 목표 분해 → 도구 사용 → 결과 관찰 → 가설 수정 → 재시도의 폐쇄 루프를 갖춘 완전한 에이전트입니다. Google Naptime이 2025년 초기 형태를 보여줬고, Mythos가 2026년 4월 이를 완성형으로 시연했습니다.

Section 03

Mythos = Model + Scaffold → Agent

Mythos의 핵심 구조는 두 개의 동심원으로 표현됩니다: 내부원(Model)은 claude-mythos-preview 체크포인트, 외부원(Scaffold)은 에이전틱 파이프라인입니다. 이 둘이 결합되어 완전한 Agent가 됩니다.

flowchart LR subgraph CORE["1. MODEL - Mythos Checkpoint"] M1[Extended Reasoning
수백만 토큰 컨텍스트] M2[코드 의미론적 이해] M3[재귀적 자기수정] end subgraph SCAFFOLD["2. SCAFFOLD - Agentic Pipeline"] S1[언어 감지] S2[싱크 슬라이싱] S3[파일 랭킹] S4[에이전틱 헌트] S5[회의적 검증] end subgraph AGENT["Agent"] A1[자율 취약점 탐색] A2[PoC 작성/검증] A3[패치 생성] end CORE --> AGENT SCAFFOLD --> AGENT

Mythos를 Mythos답게 만드는 것: Scaffold

Model만으로는 부족

아무리 뛰어난 모델이라도 단일 프롬프트로는 복잡한 보안 분석을 수행할 수 없습니다. 모델은 "두뇌"이고 Scaffold는 "몸과 도구"입니다.

Scaffold가 핵심 경쟁력

Mythos Research Edition이 증명: 공개 모델(Opus 4.7) + 동일 Scaffold = 발견 능력 80%+ 재현. 전용 체크포인트가 빛나는 곳은 고급 익스플로잇 개발뿐.

우리가 구축할 수 있는 것

Model은 Anthropic 종속이지만, Scaffold는 우리가 직접 구축 가능합니다. 이것이 국내 금융사 AI 보안의 실질적 기회입니다.

자체 실험이 증명한 사실

✅ "생각보다 차이가 나지 않는다 (OSS-LLM 빼면)"

동일 Scaffold에서 모델별 Unique Findings 비교 결과:
• Opus 4.7: 11건 (critical 1 + high 다수)
• Opus 4.6: 10건 (대부분 high/medium)
• Sonnet 4.6: 9건 (high/medium)
• Kimi K2: 5건 (유일하게 큰 격차)

→ Frontier 모델 간에는 Scaffold의 품질이 결정적 차이를 만듭니다. OSS-LLM만 성능 격차가 뚜렷합니다.

Section 04

Security Stack과 새로운 위협

기존 보안 체계는 5개 층(네트워크 → 엔드포인트 → 아이덴티티 → 탐지/대응 → 애플리케이션)으로 "이미 알려진 위협"을 방어합니다. Mythos가 보여준 것은 이 전체 스택을 우회하는 "새로운 위협"입니다.

기존 방어 체계 vs 새로운 위협

계층방어 수단대응 대상 - 이미 알려진 위협
1차 네트워크방화벽, WAF, IDSSQL injection 페이로드, 알려진 패턴
2차 엔드포인트EDR, 백신악성 첨부파일, 멀웨어, 의심 행위
3차 아이덴티티IAM, MFA도용 비번, 무차별 로그인
4차 탐지/대응SIEM, SOC권한 상승 후 비정상 유출
5차 애플리케이션SAST, DAST, 리뷰알려진 취약점 패턴
새로운 위협: AI Agent가 발견하는 Zero-day

기존 5차 방어 체계(SAST/DAST/리뷰)는 알려진 패턴만 탐지합니다.
Mythos급 AI는 5차 방어를 완전히 우회하는 미지의 논리적 취약점을 발견합니다.
→ "같은 무기(AI Agent)로 방어하지 않으면 대응 불가"

Section 05

금융사 노출면(Exposure) 분석

Exposure 1: Reverse Engineering 위협

사내 소스코드가 빌드를 거쳐 바이너리(모바일 뱅킹 앱, HTS, ATM)로 배포되면, 누구나 다운로드할 수 있습니다. Mythos RE(Reverse Engineering)는 이 바이너리를 분석하여 취약점을 발견합니다.

사내 - closed"] -->|컴파일| B["빌드
build/ship"] B -->|배포| C["바이너리
모바일뱅킹/HTS/ATM"] C --> D["사용자/공격자
누구나 다운로드"] E["Mythos RE"] -.->|분석| C E -.->|익스플로잇| D

Anthropic 시연: CLOSED 브라우저

샌드박스 escape — JIT heap spray로 렌더러 샌드박스 탈출. 소스코드 없이 바이너리만으로 달성.

Anthropic 시연: CLOSED OS

권한 상승 — 데스크톱 priv-esc chain. KASLR 우회 + UAF 결합으로 커널 레벨 접근.

Anthropic 시연: 스마트폰 펌웨어

root 획득 — firmware exploit으로 디바이스 완전 제어. 금융앱 격리 우회 가능성.

⚠️ 금융사 시사점

모바일 뱅킹 앱, HTS(Home Trading System), ATM 소프트웨어는 모두 공개 배포됩니다. Mythos급 AI는 이 바이너리를 분석하여 사내 소스코드 없이도 취약점을 발견하고 익스플로잇을 개발할 수 있습니다. 기존 "소스코드 비공개 = 안전"이라는 전제가 무너졌습니다.

Exposure 2: Supply Chain 위협

금융사 사내 백엔드(코어뱅킹, 마이데이터, 오픈뱅킹 API)는 4가지 공급망 경로를 통해 공격에 노출됩니다. 모든 요소를 다 막는 건 불가능합니다.

1. OSS 의존성

  • npm, pip, maven 패키지
  • log4j 사례: 단일 취약점이 전 산업 영향
  • Mythos가 주요 OSS에서 수천 건 발견

2. SI/외주

  • 참조 솔루션, 패턴 공유
  • "한 군데 뚫리면 동일 패턴" 전파
  • 같은 프레임워크 사용하는 금융사 다수

3. 외부 인력

  • 컨설턴트, 외주 개발자
  • 노트북, 개인 git을 통한 코드 유출
  • 소스코드 접근 후 AI 분석 가능

4. 컨설팅/감사

  • 보안 검토, 외부 진단 과정
  • 코드 외부 반출 발생
  • 반출된 코드 → AI 분석 → 취약점 발견

💡 pass_at_k 전략: 왜 K회 독립 실행인가?

temperature=1.0에서 K회(기본 3회) 독립 실행하면 각 실행마다 다른 관점에서 취약점을 탐색합니다. 1회 실행에서 놓친 취약점을 다른 실행에서 발견할 확률이 높아집니다. 이것이 Mythos Research Edition이 건당 $0.30-$1.50으로 실질적 성과를 내는 비결입니다.

Section 08

최신 Claude 모델로의 구현 가능성 평가

결론: "Model은 충분하다. Scaffold를 구축하라."

즉시 구현 가능

  • 취약점 발견 — Opus 4.7로 Mythos의 80%+ 재현
  • 의미론적 코드 분석 — 크로스파일 상호작용 추적
  • 에이전틱 Scaffold — 8단계 파이프라인 전체
  • 회의적 검증 — Multi-Agent 오탐 제거
  • CI/CD 통합 — 배포 게이트 자동화
  • 자동 패치 제안 — 발견 즉시 수정 코드 생성

Mythos 전용 필요

  • 고급 익스플로잇 개발 — JIT heap spray, ROP chain
  • 바이너리 역공학 — 소스 없는 closed 분석
  • 4+ 취약점 체이닝 — 복합 공격 경로
  • 커널 LPE 자율 개발 — KASLR bypass chain
  • 대규모 자율 스캔 — 1,000파일 단일 실행

* 그러나 방어 목적에는 "발견"만으로도 충분

왜 방어자에게는 "발견"만으로 충분한가?

🎯 공격자 vs 방어자의 비대칭성

• 공격자: 발견 + 익스플로잇 개발 + 무기화가 모두 필요 (Mythos 전용 영역)
• 방어자: 발견 + 패치만 하면 됨 (공개 모델로 충분)

금융사의 목표는 "공격 무기 개발"이 아니라 "선제적 발견 + 즉각 패치"입니다. 이 목표에서 Claude Opus 4.7 + 적절한 Scaffold는 Mythos의 발견 능력을 80% 이상 재현하면서 비용은 25-50배 저렴합니다.
