Technical Architecture Document

FSI-Mythos on AgentCore

국내 금융사를 위한 Claude 기반 자율 보안 스캐닝 플랫폼. Anthropic Mythos의 8-Phase 스캐폴드를 Strands Agents SDK와 Amazon Bedrock AgentCore로 재구현하고, Cognito·CloudFront·WAF·DynamoDB를 갖춘 풀스택으로 AWS에 배포·검증한 시스템의 전체 설계 문서.

🗓 2026-06-02 ☁️ AWS us-west-2 🤖 Claude Opus 4.8/4.7/4.6 🛡 AppSec 표준
Section 01

개요

FSI-Mythos는 소스코드에서 보안 취약점을 자율적으로 발견·검증하는 멀티에이전트 시스템입니다. 단순 패턴 매칭(SAST)을 넘어, Claude의 의미론적 추론으로 컴포넌트 간 상호작용에서 발생하는 논리적 결함까지 탐지하는 것을 목표로 합니다.

핵심 설계 원칙

  • 적대적 검증 — Hunter가 발견한 취약점을 Challenger가 반박하고 Validator가 최종 판정해 오탐(FP)을 줄임
  • 방어 목적 한정 — scan_scope="defensive", 익스플로잇 무기화 단계 미구현
  • 역할별 모델 분배 — 추론 난이도·비용에 따라 Opus 4.8/4.7/4.6 차등 배치
  • AppSec 표준 준수 — 인증·암호화·최소권한·WAF를 기본 적용
  • 재현성 — pass@k 독립 실행으로 발견의 일관성 확보

Section 03

멀티에이전트 스캐폴드

Strands Agents SDK로 4개 역할 에이전트를 구성합니다. 각 에이전트는 모델 + 시스템 프롬프트 + 도구의 조합이며, 역할별로 다른 Opus 모델과 Extended Thinking 설정을 사용합니다.

발견(Hunter)은 다양성·속도 균형의 4.7, 최종 판정(Validator)은 최고 추론력의 4.8, 비용 효율이 중요한 랭킹·반박(Ranker·Challenger)은 4.6. Challenger의 Thinking을 끈 이유는 4.6+adaptive thinking 스트리밍의 간헐적 ValidationException 회피와 비용·지연 절감 (반박 품질 영향 미미).
